Web Protocols

HTTP Methods

HTTP (Hypertext Transfer Protocol) and HTTPS (HTTP Secure) are protocols used for communication between clients (such as web browsers) and servers over the internet. They define a set of methods for requesting and responding to resources.

HTTP Methods (also used in HTTPS):

  • GET: Retrieves a resource from the server.
  • POST: Sends data to the server to create a new resource.
  • PUT: Updates a resource on the server.
  • DELETE: Removes a resource from the server.
  • HEAD: Retrieves only the headers of a resource.
  • PATCH: Partially updates a resource on the server.
  • OPTIONS: Requests information about the communication options available.

HTTP Response Status Codes

HTTP/HTTPS response status codes are three-digit numbers returned by a server to indicate the outcome of a client's request. They provide information about the success, redirection, client error, or server error related to the request. The status codes are grouped into classes, each representing a specific category of response.

Here are the classes of HTTP/HTTPS response status codes:

  • Informational (1xx): These codes indicate that the server has received the client's request and is continuing the process. Examples include 100 Continue, 101 Switching Protocols, etc.
  • Success (2xx): These codes indicate that the client's request was successfully received, understood, and processed by the server. Examples include 200 OK, 201 Created, 204 No Content, etc.
  • Redirection (3xx): These codes indicate that further action needs to be taken by the client to complete the request. They are often used for redirection purposes. Examples include 301 Moved Permanently, 302 Found, 304 Not Modified, etc.
  • Client Error (4xx): These codes indicate that the client's request contains invalid syntax or cannot be fulfilled by the server. Examples include 400 Bad Request, 403 Forbidden, 404 Not Found, 429 Too Many Requests, etc.
  • Server Error (5xx): These codes indicate that the server encountered an error while trying to fulfill a valid request from the client. Examples include 500 Internal Server Error, 503 Service Unavailable, 504 Gateway Timeout, etc.

DNS

DNS (Domain Name System) records are used to map human-readable domain names to specific IP addresses or to provide other information about a domain. Here are the different types of DNS records:

  • A (Address) Record: Maps a domain name to an IPv4 address.
  • AAAA (IPv6 Address) Record: Maps a domain name to an IPv6 address.
  • CNAME (Canonical Name) Record: Creates an alias or canonical name for an existing record. It maps a domain name to another domain name.
  • MX (Mail Exchanger) Record: Specifies the mail servers responsible for accepting incoming email for a domain. It identifies the email server for the domain.
  • NS (Name Server) Record: Specifies the authoritative name servers for a domain. It indicates which servers are responsible for resolving the domain name.
  • PTR (Pointer) Record: Maps an IP address to a domain name. It is the reverse of an A or AAAA record.
  • TXT (Text) Record: Allows domain owners to add arbitrary text to the DNS record. It can be used for various purposes, such as domain verification or adding descriptive text.
  • SRV (Service) Record: Specifies the location of services or servers for a particular protocol and domain. It provides information about available services, such as SIP, LDAP, or XMPP.
  • SPF (Sender Policy Framework) Record: Specifies the authorized mail servers for a domain. It helps prevent email spoofing and verifies the email sender's authenticity.
  • DKIM (DomainKeys Identified Mail) Record: Provides a digital signature associated with an email message, ensuring its authenticity and integrity.
  • SOA (Start of Authority) Record: Contains administrative information about a DNS zone, including the primary name server, contact email address, and other details.

How to determine IP address

To determine the IP address of a domain, such as google.com, google.com.ua, or ucode.world, you can use various methods:

  1. Using the Command Prompt (Windows) or Terminal (Mac/Linux):
    1. Open the Command Prompt (Windows) or Terminal (Mac/Linux).
    2. Type the command nslookup followed by the domain name. For example:
      • nslookup google.com
      • nslookup google.com.ua
      • nslookup ucode.world
    3. Press Enter.
    4. The IP address associated with the domain will be displayed.
  2. Using the ping Command:
    1. Open the Command Prompt (Windows) or Terminal (Mac/Linux).
    2. Type the command ping followed by the domain name. For example:
      • ping google.com
      • ping google.com.ua
      • ping ucode.world
    3. Press Enter.
    4. The IP address associated with the domain will be displayed along with other information.
  3. Using Online Tools:
    1. Various online tools are available that allow you to find the IP address of a domain. Examples include "WhatIsMyIP.com," "IP Location Lookup," or "DNS Lookup" tools.
    2. Open a web browser.
    3. Visit one of these online tools.
    4. Enter the domain name in the provided field or select the appropriate options.
    5. Click on the "Lookup" or "Search" button.
    6. The IP address associated with the domain will be displayed.
